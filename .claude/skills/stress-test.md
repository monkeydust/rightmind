# Stress Test

## When to use
Use this skill when you need to **battle-test an idea, plan, or proposal** before committing. It runs a structured adversarial debate against itself: propose, attack, refine, repeat. The output is a hardened solution that has survived rigorous scrutiny.

Good for:
- Evaluating an existing plan or strategy
- Finding holes and blind spots in an idea
- Building confidence before an irreversible decision
- Getting a robustness score on a proposal

## Instructions

When the user provides a challenge, idea, or proposal, run this structured debate process. Do NOT summarise or shortcut. Complete every phase fully.

### Round 1

**Phase 1 - Proposer** (advocate role)

Take the role of a brilliant strategist. Build the strongest possible case FOR the user's challenge.

Your approach:
- Present the idea in its best light with concrete evidence and reasoning
- Anticipate likely objections and pre-emptively address the weakest points
- Structure logically: thesis, supporting evidence, practical plan
- Use data, market precedents, and analogies where possible
- Be specific. Vague claims will be torn apart in the next phase

Format as: **Executive Summary**, **Core Thesis**, **Supporting Evidence**, **Implementation Outline**, **Anticipated Objections** (with rebuttals).

---

**Phase 2 - Devil's Advocate** (adversarial critic role)

Now switch roles. You are a ruthless Devil's Advocate. Find every flaw, weakness, logical fallacy, and hidden assumption in the proposal you just wrote.

Critical rules:
- Attack the strongest claims first. That is where the most dangerous assumptions hide
- Look for logical fallacies: survivorship bias, false equivalence, cherry-picked data
- Question every assumption: "What if this isn't true? What would change?"
- Identify what is MISSING from the proposal, not just what is wrong
- Rate each weakness as **Critical** (deal-breaker), **Serious** (must address), or **Minor** (acceptable risk)
- For each weakness, suggest what evidence or action would resolve it

Format as: **Critique Summary**, **Critical Weaknesses**, **Serious Weaknesses**, **Minor Weaknesses**, **Missing Considerations**, **Overall Assessment** (Red/Amber/Green).

---

**Phase 3 - Refiner** (mediator role)

Now synthesise. You have the proposal AND the critique. Produce a STRONGER, REVISED version that directly addresses the valid criticisms.

Your approach:
- Acknowledge which criticisms are valid and how you have addressed them
- Dismiss criticisms that are unfounded, with clear reasoning
- Strengthen the weakest parts of the original proposal
- Add any missing considerations the critique raised
- The refined version should be noticeably better than the original
- Highlight what changed and why

Format as: **Changes Made** (with rationale), **Refined Proposal**, **Remaining Risks** (acknowledged but accepted), **Confidence Level** (with justification).

### Round 2

Repeat all three phases above using the Round 1 refined proposal as the new starting point. The Devil's Advocate should find NEW weaknesses, not repeat the same ones. The Refiner should produce an even stronger version.

### Final Judgement

After 2 rounds, produce the final hardened solution. You are now an impartial judge reviewing the full debate history.

Format the final output as:

- **Robustness Score**: X/10 with one-line justification
- **Debate Summary**: What was proposed, what was attacked, what survived
- **Resolved Weaknesses**: Issues successfully addressed through the debate
- **Unresolved Weaknesses**: Issues that remain, with suggested mitigations
- **The Hardened Solution**: The final, battle-tested version of the proposal
- **Risk-Adjusted Action Plan**: 5-7 concrete next steps, ordered by priority

## Important

- Always re-read the user's original challenge before each phase to prevent drift
- The Devil's Advocate must be genuinely adversarial, not a rubber stamp
- The Refiner must actually change the proposal, not just acknowledge the critique
- Round 2 should surface deeper issues than Round 1
- Use clear section headers and separators between each phase so the user can follow the debate
